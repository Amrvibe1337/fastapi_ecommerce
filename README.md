# E-commerce API на FastAPI

Добро пожаловать в учебный проект **E-commerce API**!  
Это backend-приложение для интернет-магазина, построенное на современном стеке технологий с упором на асинхронность, безопасность и чёткое разграничение прав доступа.

## Описание
Разработал API интернет-магазина на **FastAPI**, используя **асинхронную работу** с базой данных **PostgreSQL** через `AsyncSession`, **SQLAlchemy** и драйвер `asyncpg`. 
Проект охватывает ключевые функциональности для управления пользователями и товарами с категориями, а также отзывами для товаров.

## Стек технологий

- **FastAPI** — современный, быстрый веб-фреймворк для создания API.
- **PostgreSQL** + драйвер **asyncpg** — асинхронная работа с базой данных.
- **SQLAlchemy 2.0** (с поддержкой `AsyncSession`) — асинхронный ORM для взаимодействия с БД.
- **Alembic** — управление миграциями.
- **JWT** — аутентификация и выдача токенов доступа.
- **Pydantic** — валидация данных и работа со схемами.

## Требования

- Python 3.8+
- Virtual environment (рекомендуется)

## Основные возможности

### 1. Регистрация и аутентификация
- Регистрация пользователей с ролями: **admin** (админ), **seller** (продавец) и **buyer** (покупатель) — `POST /users/`
- Вход в систему и получение JWT токенов:
  - `POST /users/token`
  - `access_token` (действителен 30 минут)
  - `refresh_token` (действителен 7 дней)
- Аутентификация через `OAuth2PasswordBearer` (логин и пароль)
- Проверка статуса `is_active` для всех запросов, требующих авторизации

### 2. Управление товарами и категориями
- Полноценные CRUD операции для товаров и категорий (`/products/`, `/categories/`)
- Привязка товаров к конкретному продавцу (`seller_id`)
- Доступ к созданию товаров только для пользователей с ролью **seller**
- Доступ к созданию категорий только для администраторов (роль **admin**)

### 3. Отзывы на товары
- Модель отзывов (`Review`), привязанная к товарам
- Возможность оставлять отзывы только для авторизованных **buyer**
- Каждый отзыв привязывается к конкретному покупателю (`buyer_id`)
- Реализована функция (`update_product_rating`) для пересчёта рейтинга о товарах при их добавлении и удалении

### 4. Обновление токенов
- Эндпоинт `POST /refresh-token` — получение нового `refresh_token`
- Эндпоинт `POST /access-token` — продление `access_token` при наличии актуального `refresh_token`

### 5. Безопасность
- Все JWT токены подписываются с использованием ключа `SECRET_KEY` и алгоритма `HS256`
- Проверка роли пользователя через зависимости `get_current_seller`, `get_current_buyer` и `get_current_admin`
- Валидация активности пользователя (`is_active`) при каждом запросе

## Установка и запуск

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/AmirHan007/fastapi_ecommerce.git
   cd fastapi_ecommerce
   ```
2. Создайте и активируйте виртуальное окружение:
  - Для Linux/Mac:
    ```bash
    source .venv/bin/activate
    ```
  - Для Windows:
    ```bash
    .venv\Scripts\activate
    ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Создайте файл .env на основе .env.example и отредактируйте его:
   ```bash
   cp .env.example .env
   ```
   Измените DATABASE_URL и SECRET_KEY.
   Для создания своего `SECRET_KEY` надо:
   ```bash
   openssl rand -hex 32
   ```
5. Примените миграции:
   ```bash
   alembic upgrade head
   ```
6. Запуск через Uvicorn:
   ```bash
   uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
   ```
   Приложение будет доступно по адресу: http://localhost:8000
